# Comprehensive PR Review

包括的PRレビュー - 10エージェント統合実行（2 Wave分割）

## 概要

pr-review-toolkit（品質観点）とCEK code-review（防御観点）を統合し、10エージェントによる包括的なPRレビューを実行します。

**メモリ効率**: 最大5エージェント並列 × 2 Wave で実行（PCメモリ枯渇防止）

## 使用タイミング

- 重要なPR作成時
- リリース前のコードレビュー
- セキュリティ影響のある変更時

## 実行フロー（2 Wave分割）

### Wave 1: 品質観点（5エージェント並列）

**実行コマンド**: `/pr-review-toolkit:review-pr`

| エージェント | 役割 |
|-------------|------|
| code-reviewer | コード品質・設計パターン |
| comment-analyzer | コメント正確性・ドキュメント |
| type-design-analyzer | 型設計・インターフェース |
| silent-failure-hunter | サイレント失敗・エラーハンドリング |
| pr-test-analyzer | テストカバレッジ・品質 |

### Wave 2: 防御観点 + 簡素化（5エージェント並列）

**実行コマンド**: `/code-review:review-pr`

| エージェント | 役割 |
|-------------|------|
| code-simplifier | コード簡素化・リファクタリング提案 |
| Bug Hunter | 競合状態・メモリリーク・ヌルポ検出 |
| Security Auditor | SQLi・XSS・認証回避・API認証検証 |
| Contracts Reviewer | API互換性・スキーマ検証・後方互換性 |
| Historical Context | 技術債・アーキテクチャ一貫性 |

## 実行手順

```
┌─────────────────────────────────────────────────┐
│ Wave 1: 品質観点（5エージェント並列）            │
│ → /pr-review-toolkit:review-pr                  │
│ → 結果確認・緊急問題対応                         │
└─────────────────────────────────────────────────┘
                    ↓ 完了後
┌─────────────────────────────────────────────────┐
│ Wave 2: 防御観点（5エージェント並列）            │
│ → /code-review:review-pr                        │
│ → 結果統合・優先度整理                           │
└─────────────────────────────────────────────────┘
```

### 詳細手順

1. **Wave 1 実行**:
   ```
   /pr-review-toolkit:review-pr
   ```
   - 5エージェントが並列実行
   - Critical/High問題があれば先に対応

2. **Wave 1 完了確認** → メモリ解放を待つ（数秒）

3. **Wave 2 実行**:
   ```
   /code-review:review-pr
   ```
   - 5エージェントが並列実行
   - セキュリティ・バグ・API契約を検証

4. **結果統合**: 両Waveの結果を統合し、優先度順にアクション項目を整理

## 判定基準

- **Critical**: セキュリティ脆弱性、データ損失リスク → 即時修正必須
- **High**: バグ、API互換性問題 → マージ前に修正
- **Medium**: コード品質、テストカバレッジ → 可能な限り対応
- **Low**: 簡素化提案、スタイル → 次回対応可

## 日常レビューとの使い分け

| シーン | 推奨ツール | エージェント数 | Wave数 |
|--------|-----------|---------------|--------|
| 日常開発 | `/code-review:code-review` | 4 | 1 |
| PR作成時 | `/comprehensive-pr-review` | 10 | 2 |
| 緊急修正 | `/code-review:code-review` | 4 | 1 |

## 注意事項

- **メモリ効率**: 5エージェント × 2 Wave で実行（同時10エージェント回避）
- **待機時間**: Wave 1完了後、数秒待ってからWave 2を実行
- **トークン消費**: 10エージェント実行は大きいため、重要PR時のみ使用
- 日常レビューには `/code-review:code-review` を使用
