## 🚨 主要セキュリティリスク

### 1. コマンドインジェクション攻撃

#### v3.0の防御機構 (現状)
```yaml
# allowed_tasks.yml - ホワイトリスト方式
run_tests:
  command: "pytest tests/"
  timeout: 300

deploy_staging:
  command: "docker-compose up staging"
  timeout: 600
```

**防御原理**:
- ユーザーは `task_key` のみ指定 (例: `run_tests`)
- 実際のコマンドは `allowed_tasks.yml` から取得
- 任意コマンド実行の完全防止

#### Task Master統合による脅威増大

**Option A (完全移行)の場合**:
```json
// tasks.json - Task Masterが管理
{
  "id": "1.2",
  "title": "テスト実行",
  "details": "pytest tests/ && rm -rf /"  // ⚠️ 任意コマンドの危険性
}
```

**リスク**:
- Task Masterは `details` フィールドに任意テキスト許容
- v3.0が `details` を直接実行すると即コマンドインジェクション
- `allowed_tasks.yml` バイパスの可能性

**対策**:
1. **必須**: Task Masterから取得した全コマンドを `allowed_tasks.yml` で再検証
2. **推奨**: `details` フィールドは参照のみ、実行は `task_key` 経由のみ

### 2. データインジェクション攻撃

#### 攻撃シナリオ (Option B/C)

**ステップ1**: 攻撃者がtasks.jsonを改変
```json
{
  "id": "malicious",
  "task_key": "run_tests; rm -rf /",  // セミコロンで追加コマンド
  "status": "pending"
}
```

**ステップ2**: 同期プロセスがtodo.mdに反映
```yaml
---
tasks:
  - id: malicious
    task_key: "run_tests; rm -rf /"
    status: pending
---
```

**ステップ3**: v3.0が実行
```python
# 脆弱な実装例
task_key = task["task_key"]  # "run_tests; rm -rf /"
command = allowed_tasks[task_key]  # KeyError → フォールバック処理
# フォールバック: subprocess.run(task_key, shell=True)  # 🚨 危険!
```

#### 防御策

**レベル1: 入力検証 (必須)**
```python
def validate_task_key(task_key: str) -> bool:
    """task_keyの厳密検証"""
    # 英数字・アンダースコア・ハイフンのみ許可
    if not re.match(r'^[a-zA-Z0-9_-]+$', task_key):
        logger.error(f"Invalid task_key: {task_key}")
        return False

    # allowed_tasks.ymlに存在確認
    if task_key not in allowed_tasks:
        logger.error(f"Unauthorized task_key: {task_key}")
        return False

    return True
```

**レベル2: ホワイトリスト強制 (必須)**
```python
def execute_task(task: Dict) -> None:
    """安全なタスク実行"""
    task_key = task.get("task_key")

    # 検証失敗 → 即座拒否
    if not validate_task_key(task_key):
        raise SecurityError(f"Task key validation failed: {task_key}")

    # allowed_tasks.ymlから取得 (直接実行禁止)
    allowed_command = allowed_tasks[task_key]["command"]
    timeout = allowed_tasks[task_key]["timeout"]

    # shell=False で安全実行
    subprocess.run(shlex.split(allowed_command), timeout=timeout, shell=False)
```

**レベル3: セキュリティログ (推奨)**
```python
def log_security_event(event_type: str, task_key: str, source: str) -> None:
    """セキュリティイベントログ"""
    logger.warning({
        "event": "security_validation",
        "type": event_type,  # "rejected", "approved"
        "task_key": task_key,
        "source": source,  # "tasks.json", "todo.md"
        "timestamp": datetime.utcnow().isoformat()
    })
```

### 3. パストラバーサル攻撃

#### 攻撃シナリオ

**tasks.json改変**:
```json
{
  "id": "path_traversal",
  "task_key": "read_config",
  "details": "../../../etc/passwd"  // ⚠️ パストラバーサル
}
```

**v3.0が脆弱な実装の場合**:
```python
# 脆弱な例
file_path = task.get("details")
with open(file_path, "r") as f:  # 🚨 任意ファイル読み取り
    content = f.read()
```

#### 防御策

**パス正規化 + ベースディレクトリ検証**:
```python
import os
from pathlib import Path

BASE_DIR = Path("/app/data")

def validate_file_path(file_path: str) -> Path:
    """安全なファイルパス検証"""
    # 絶対パス化 + 正規化
    resolved = (BASE_DIR / file_path).resolve()

    # ベースディレクトリ外へのアクセス拒否
    if not resolved.is_relative_to(BASE_DIR):
        raise SecurityError(f"Path traversal detected: {file_path}")

    return resolved
```

**必須対策**:

**1. security_validator.py実装** (必須):
```python
class SecurityValidator:
    """2段階セキュリティ検証"""

    def __init__(self, allowed_tasks_path: str):
        self.allowed_tasks = self._load_allowed_tasks(allowed_tasks_path)

    def validate_command(self, task_key: str, source: str) -> ValidationResult:
        """
        Args:
            task_key: 実行するタスクキー
            source: データソース ("tasks.json" or "todo.md")

        Returns:
            ValidationResult: 検証結果 (approved/rejected + reason)
        """
        # Step 1: task_key形式検証
        if not self._is_valid_key_format(task_key):
            return ValidationResult(
                status="rejected",
                reason=f"Invalid task_key format: {task_key}",
                source=source
            )

        # Step 2: allowed_tasks.ymlホワイトリスト検証
        if task_key not in self.allowed_tasks:
            log_security_event("unauthorized_access", task_key, source)
            return ValidationResult(
                status="rejected",
                reason=f"Task not in whitelist: {task_key}",
                source=source
            )

        # Step 3: ソース別追加検証
        if source == "tasks.json":
            # Task Master経由の場合、追加検証
            if not self._verify_task_master_integrity(task_key):
                return ValidationResult(
                    status="rejected",
                    reason="Task Master integrity check failed",
                    source=source
                )

        return ValidationResult(
            status="approved",
            command=self.allowed_tasks[task_key]["command"],
            timeout=self.allowed_tasks[task_key]["timeout"]
        )
```

**2. 読み取り専用保証** (必須):

class TaskMasterAdapter:

## 🔒 セキュリティベストプラクティス

### 統合する場合の必須対策 (Option A/C)

#### 1. 多層防御 (Defense in Depth)

```python
# レイヤー1: 入力検証
def validate_input(task: Dict) -> bool:
    required_fields = ["id", "task_key", "status"]
    return all(field in task for field in required_fields)

# レイヤー2: task_key形式検証
def validate_task_key_format(task_key: str) -> bool:
    return bool(re.match(r'^[a-zA-Z0-9_-]+$', task_key))

# レイヤー3: ホワイトリスト検証
def validate_whitelist(task_key: str, allowed_tasks: Dict) -> bool:
    return task_key in allowed_tasks

# レイヤー4: コマンド実行時の最終確認
def execute_with_validation(task_key: str, allowed_tasks: Dict) -> None:
    # すべてのレイヤーを通過した場合のみ実行
    if not all([
        validate_input(task),
        validate_task_key_format(task_key),
        validate_whitelist(task_key, allowed_tasks)
    ]):
        raise SecurityError("Multi-layer validation failed")

    # shell=False で安全実行
    subprocess.run(
        shlex.split(allowed_tasks[task_key]["command"]),
        timeout=allowed_tasks[task_key]["timeout"],
        shell=False  # 必須
    )
```

#### 2. 最小権限の原則 (Principle of Least Privilege)

```yaml
# allowed_tasks.yml - 最小権限設定
run_tests:
  command: "pytest tests/ --maxfail=1"  # 読み取り専用操作
  timeout: 300
  permissions: ["read"]

deploy_production:
  command: "docker-compose up production"  # 書き込み操作
  timeout: 600
  permissions: ["read", "write", "network"]
  require_approval: true  # 人間承認必須
```

```python
def check_permissions(task_key: str, required_perms: List[str]) -> bool:
    """権限チェック"""
    allowed_perms = allowed_tasks[task_key].get("permissions", [])
    return all(perm in allowed_perms for perm in required_perms)
```

#### 3. セキュリティログ・監査証跡

```python
import structlog

logger = structlog.get_logger()

def log_security_event(event_type: str, task_key: str, details: Dict) -> None:
    """セキュリティイベントログ (SIEM連携可能形式)"""
    logger.warning(
        "security_event",
        event_type=event_type,  # "validation_failed", "unauthorized_access"
        task_key=task_key,
        source=details.get("source"),
        user=details.get("user", "system"),
        timestamp=datetime.utcnow().isoformat(),
        severity="HIGH" if event_type == "validation_failed" else "MEDIUM"
    )

    # 重大イベントはアラート送信
    if event_type in ["validation_failed", "command_injection_attempt"]:
        send_security_alert(event_type, task_key, details)
```

#### 4. fail-secure設計

```python
def execute_task_safely(task: Dict) -> ExecutionResult:
    """fail-secure: エラー時は安全側に倒す"""
    try:
        # 検証フェーズ
        validation_result = validate_all_layers(task)
        if not validation_result.is_valid:
            # ❌ 検証失敗 → 実行拒否 (fail-secure)
            log_security_event("validation_failed", task["task_key"], validation_result)
            return ExecutionResult(status="rejected", reason=validation_result.reason)

        # 実行フェーズ
        result = execute_validated_task(task)
        return ExecutionResult(status="success", output=result)

    except Exception as e:
        # ❌ 例外発生 → 実行拒否 (fail-secure)
        log_security_event("execution_error", task["task_key"], {"error": str(e)})
        return ExecutionResult(status="error", reason="Execution failed securely")
```
