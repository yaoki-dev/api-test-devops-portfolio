# Task Masterで完了タスクをtask.jsonに残し続けるとファイルサイズが大きくなり、未実行タスクの検索が困難になるため、完了タスクを別ファイルに自動で切り出して管理するのは有効な運用方法です。

## ベストプラクティスの運用方法
### 1. 完了したタスクはtask.jsonから自動的に切り出し、完了済みタスク専用のファイル（例：completed_tasks.json）に移動して管理することで、未完タスクの管理性を維持する。

### 2. task.jsonは未実行・進行中のタスクのみを保持し、軽量化して高速に検索可能にする。

### 3. 完了タスクの履歴は別ファイルまたはログ形式で保存し、後から参照・集計・分析できるようにする。

### 4. タスクのステータス変更時に、切り出し処理を自動化して手動ミスを防止する。

### 5. タスク全体の一元管理は、未完タスクと完了タスクのファイルを連携して閲覧・検索できるツールやUIを用意する。

## 実装詳細例

### 1. タスクの状態変更でフックを設定：
- タスクが完了（status = "done"）になった時点でイベントを検知

### 2. 完了タスクをtask.jsonから削除し、completed_tasks.jsonに追加：
- JSONの読み書き処理を行い、task.jsonは未完のタスクのみを保持
- completed_tasks.jsonには完了タスクを追記形式で保存

#### 3. ファイル操作は定期的または即時に実行できるよう、タスク管理システムのAPIやCLIコマンドに組み込む

#### 4. 検索は未完タスク用のtask.jsonを対象にし、必要に応じて完了タスクファイルも参照可能に

#### 5. バックアップやアーカイブ設定を加え、ファイル肥大化防止や履歴管理を強化
