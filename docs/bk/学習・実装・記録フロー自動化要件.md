  要件まとめ

  1.1 ビジネス要件（Why）

  課題:
  - Week 1-6の学習進捗とポートフォリオ実装の記録が手動で非効率
  - 日次・週次の振り返りに時間がかかる（30-60分/回）
  - 学習計画（8-10週ハイブリッドプラン）との乖離が見えづらい
  - AI協働率・自律度の定量的トラッキングが困難

  目標:
  - 学習記録時間を90%削減: 30分 → 3分
  - 実装記録時間を95%削減: 20分 → 1分（「実装記録」入力のみ）
  - 週次振り返り時間を70%削減: 60分 → 18分（自動集計）
  - 進捗可視化: ポートフォリオメトリクスのリアルタイム更新

  1.2 機能要件（What）

  必須機能（Phase 1）

  | 機能ID | 機能名       | 説明                            | 優先度      |
  |------|-----------|-------------------------------|----------|
  | F1   | 学習開始ガイド   | 現在のWeek/Day/Phase表示、推奨学習タスク提示 | Critical |
  | F2   | 実装開始ガイド   | ポートフォリオメトリクス表示、進行中タスク提示       | Critical
  |
  | F3   | 学習記録自動化   | 学習時間・内容・課題の構造化記録              | Critical |
  | F4   | 実装記録完全自動化 | Git履歴から実装内容・時間・メトリクス自動抽出      | Critical
  |
  | F5   | 週次振り返り自動化 | 日次記録集計、週次サマリー自動生成             | High     |

  推奨機能（Phase 2 - Optional）

  | 機能ID | 機能名       | 説明                 | 優先度    |
  |------|-----------|--------------------|--------|

      - "volume設定エラー"
      - "build context最適化未理解"

      name: "docker-compose.yml 4環境実装学習"

  2.3 トリガー仕様

  トリガー1: 学習開始

  検出キーワード: "学習開始", "学習スタート", "今日の学習"

  実行フロー:
  1. Read learning_state.yaml → 現在状態取得
  2. Read 8-10週ハイブリッドプラン → Week 7 Day 3セクション抽出
  3. 提示:
     📍 現在位置: Week 7 Day 3 - Phase A (45%完了)
     ⚠️ 前回の課題: [issues リスト]
     🎯 推奨タスク: [next_learning_task]
     
     選択肢:
     a) 推奨タスク開始
     b) 前回課題復習
     c) 週次振り返り
     d) 学習計画調整

  所要時間: 10秒

  トリガー2: 実装開始

  検出キーワード: "実装開始", "コーディング開始", "ポートフォリオ作業開始"

  実行フロー:
  1. Read learning_state.yaml → portfolio.current_tasks取得
  2. Read ポートフォリオ戦略分析.md → Week 7実装ロードマップ抽出
  3. 提示:
     🛠️ ポートフォリオ実装セッション開始
     📊 現在メトリクス: [metrics一覧]
     🎯 進行中タスク: [in_progress tasks]
     
     選択肢:
     a) 進行中タスク継続
     b) 新規タスク開始
     c) 実装進捗確認
     d) 品質ゲート実行

  所要時間: 10秒

  トリガー3: 学習記録

  検出キーワード: "学習記録", "今日の学習記録"

  実行フロー:
  1. 構造化入力テンプレート提示:
     📝 学習記録入力:
     - 学習時間: ___h
     - 学習内容: ______
     - 課題・ブロッカー: ______
     - 翌日予定: ______
     - AI協働率: ___%（推定でOK）

  2. Edit learning_state.yaml:
     - learning.previous_issues 更新
     - learning.next_learning_task 更新（AI推論）
     - learning.last_updated 更新
     - current_week_goals.learning_hours_actual += 入力時間

  3. Edit daily_progress.md:
     - 日次学習進捗セクション追加

  所要時間: 2分（ユーザー入力1.5分 + AI処理30秒）

  トリガー4: 実装記録（完全自動化） ★

  検出キーワード: "実装記録", "今日の実装記録", "ポートフォリオ記録"

  実行フロー:
  1. 自動検出（並列実行）:
     Bash: git log -1 --pretty=format:"%s"    # コミットメッセージ
     Bash: git log -1 --pretty=format:"%ai"   # コミット時刻
     Bash: git diff --stat HEAD~1 HEAD        # 差分統計
     Bash: git log -1 --pretty=format:"%h"    # ハッシュ
     Bash: uv run pytest --cov | grep "TOTAL" # カバレッジ
     Bash: uv run ruff check .                # Lint
     Bash: uv run mypy utils/ config/         # Type check

  2. AI推論:
     - Read learning_state.yaml → current_tasks取得
     - コミットメッセージ + 差分 → 実装内容要約生成
     - コミット時刻 vs last_implementation.date → 実装時間推定
     - 要約 + 差分ファイル → タスクマッチング
     - キーワード分析（"完了"/"実装"） → progress推定
     - pytest出力 → test_coverage抽出
     - タスク完了 → ポートフォリオ戦略分析参照 → metrics算出

  3. 確認プロンプト提示:
     🤖 実装記録自動検出結果:
     📝 実装内容: [AI生成要約]
     ⏱️ 実装時間: [推定]h
     ✅ 完了タスク: [task_id] ([progress]%)
     📊 メトリクス変化: [詳細]
     🔍 品質ゲート: pytest/ruff/mypy結果
     
     a) そのまま記録
     b) 実装時間修正
     c) 内容修正
     d) キャンセル

  4. 更新実行（選択肢 a の場合）:
     Edit learning_state.yaml:
     - portfolio.metrics.* 更新
     - current_tasks[].status/progress 更新
     - last_implementation 更新
     - current_week_goals.implementation_hours_actual += 推定時間
     
     Edit daily_progress.md:
     - 日次実装進捗セクション追加

  所要時間: 30秒（全自動 + ユーザー確認10秒）

  トリガー5: 週次振り返り

  検出キーワード: "週次振り返り", "今週の振り返り", "週次レポート"

  実行フロー:
  1. Read learning_state.yaml → 全状態取得
  2. Read daily_progress.md → Week 7全日次記録抽出
  3. AI分析:
     - 総学習時間・実装時間集計
     - 完了タスク一覧
     - メトリクス変化集計
     - 目標達成度算出
     - 課題・ブロッカー頻度分析
     - AI推奨アクション生成

  4. 提示:
     📊 Week 7 振り返りレポート:
     [学習進捗サマリー]
     [実装進捗サマリー]
     [目標達成度]
     [主要課題]
     [AI推奨アクション]

  5. Edit daily_progress.md:
     - Week 7サマリーセクション自動生成・挿入

  6. Read 8-10週ハイブリッドプラン → Week 8目標抽出

  7. 提示:
     🎯 Week 8 目標設定案:
     [学習時間目標]
     [実装時間目標]
     [主要成果物]
     
     この目標でWeek 8を開始しますか？ (y/n)

  8. Edit learning_state.yaml（y の場合）:
     - current_week: 8
     - current_day: 1
     - current_week_goals 更新

  所要時間: 3分（AI分析2分 + ユーザー確認1分）
