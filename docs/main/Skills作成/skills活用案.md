## 導入方法

Claude Codeにおけるスキルは、以下の2通りの場所に配置できます。

*   **グローバル（個人）スキル**: `~/.claude/skills/`
*   **プロジェクトスキル**: `.claude/skills/` (リポジトリ内に作成)

**カスタムスキルの基本的なディレクトリ構造:**


```
.claude/
└── skills/
    └── my-custom-skill/        # Skill名
        ├── SKILL.md            # スキルのメタデータと指示内容を定義
        ├── helper-script.py    # スキルの具体的なロジックを実装するスクリプト
        └── config.yaml         # スキル固有の設定ファイル (例: ドキュメント依存関係ルール)
```

この構造でスキルを定義すれば、Claude Codeが自動的にスキャンし、必要に応じて読み込みます。

### `SKILL.md` の役割
`SKILL.md` ファイルには、スキルの名前、説明、そしてAIエージェントがスキルを実行する際の具体的な手順をYAML形式のメタデータとMarkdownで記述します。

**`helper-script.py` の役割:**
`helper-script.py` は、スキルの具体的なロジックを実装するPythonスクリプトです。`SKILL.md` から呼び出され、引数を受け取り、処理結果を標準出力に返します。例えば、ドキュメントの依存関係を解析し、関連するドキュメントを提案する処理を記述します。

**設定ファイル (`config.yaml` など) の活用:**
特に「ドキュメント関連性検出と更新提案」のような自作スキルでは、`config.yaml` などの設定ファイルが重要です。このファイルには、`docs/` ディレクトリ構造を活用したドキュメント間の依存関係ルール（例: 特定のディレクトリ内の変更が他のどのドキュメントに影響するか、明示的なファイル間の関連性など）を定義します。これにより、スキルはプロジェクト固有の知識に基づいて、より正確な提案を行うことができます。
