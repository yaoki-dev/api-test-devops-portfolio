## 🤖 自動化スコープ（何が自動で、何が手動か）

### ✅ AI自動処理

- **Git解析**: commit hash、変更行数、変更ファイル数の自動抽出
- **品質ゲート実行**: pytest/ruff/mypy の自動実行（4つのGate）
- **commit message生成**: AI解析によるConventional Commits自動生成
- **メトリクス抽出**: カバレッジ、テスト数の自動計算
- **習熟度計算**: 学習時間から自動算出（目標80%基準）
- **YAML/MD更新**: progress_state.yaml、daily_progress.mdの自動更新

### 🖊️ ユーザー入力必須

- **git add .**: 実装ファイルのステージング（手動）
- **commit message承認**: AI生成メッセージの確認・承認・修正（3択）
- **学習時間**: 実際の学習時間の入力（例: 5h）
- **課題・ブロッカー**: つまずいた点の記述（任意）

### ⏱️ 実行時間の目安

- 学習記録: 1分（ユーザー入力30秒 + AI処理30秒）
- 実装記録: 30-60秒（品質ゲート実行含む、完全自動）
- 理解度確認: 15分（AI生成問題 + ユーザー回答）
- 週次振り返り: 2分（集計30秒 + AI生成90秒）

### 実装のみの日

**前提条件**: `git add .`でファイルをステージング済み

Claude Codeに以下を伝えてください：

```
実装記録
```

**AI自動実行内容**（30-60秒）:

1. 品質ゲート4つ自動実行:
   - Gate 1: `pytest --cov-fail-under=[目標]`
   - Gate 2: `ruff check .`
   - Gate 3: `mypy utils/ config/ models/`
   - Gate 4: ステージング状態確認
2. commit message自動生成（Conventional Commits形式）
3. ユーザー承認プロンプト表示（承認/修正/中止）
4. 承認後: `git commit`自動実行
5. メトリクス抽出・YAML/MD自動更新

**💡 補足**: 上記は初心者向けの簡略版フローです。詳細な実装フロー（Step 0-9）は`フロー自動化改善要件.md`の「トリガー4: 実装記録」セクションを参照してください。

**💡 Gate失敗時の対処**:

- **Gate 1失敗** → `uv run pytest -vv` で詳細確認、テスト修正後に再実行
- **Gate 2失敗** → `uv run ruff check --fix .` で自動修正、再実行
- **Gate 3失敗** → 型ヒント追加、`uv run mypy --strict` で確認後に再実行
- **Gate 4失敗** → `git add .` 実行、未追跡ファイルを `git status` で確認

**💡 pre-commit hookとの違い**:

- **日常開発**: `git commit` → ruffのみ（< 3秒、高速）
- **実装記録**: 全チェック実行（30-60秒、品質保証）
