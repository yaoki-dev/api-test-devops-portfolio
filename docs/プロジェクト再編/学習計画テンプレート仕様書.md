# 学習計画テンプレート仕様書

*最終更新: 2025年10月31日*

## 概要

10週ハイブリッドプランは、学習目的と週次進捗に応じて**5種類の異なるテンプレート形式**を使用します。これは設計上の意図的な変化であり、テンプレート欠落ではありません。

## テンプレート分類マップ

### Template Type 1: Full Standard Template（完全標準テンプレート）

**適用範囲**: Week 1 (D1-6)
**目的**: AI協働学習の基礎教育、AI依頼品質向上

**必須要素（12個）**:
- ✅ day_header
- ✅ total_learning_time
- ✅ coverage_goal
- ✅ phase1: AI説明・概念理解
- ✅ understanding_checklist (5項目チェックボックス)
- ✅ judgment_criteria (判定基準)
- ✅ criteria_3plus (3つ以上✅基準)
- ✅ phase2: AI協働実装
- ✅ ai_request_prompt (AI依頼プロンプトセクション)
- ✅ phase3: 理解度確認・記録
- ✅ review_buffer
- ✅ portfolio_deliverables

**検出率 (Week 1平均)**:
- AI依頼プロンプト: 100%
- 理解度チェックリスト: 100%
- 判定基準: 100%
- 3つ以上✅: 83%

**教育意図**: Week 1では、AI協働の基礎（プロンプト品質、理解度自己評価）を徹底教育するため、全要素を含む最も詳細なテンプレートを使用。

---

### Template Type 2: Standard Template (No AI Prompt)（標準テンプレート・AI依頼セクションなし）

**適用範囲**: Week 2-4 (D7-24)
**目的**: 自律的学習への移行、AI依存度低減

**必須要素（11個）**:
- ✅ day_header
- ✅ total_learning_time
- ✅ coverage_goal
- ✅ phase1: AI説明・概念理解
- ✅ understanding_checklist (5項目チェックボックス)
- ✅ judgment_criteria (判定基準)
- ✅ criteria_3plus (3つ以上✅基準)
- ✅ phase2: AI協働実装
- ❌ ai_request_prompt (意図的に削除)
- ✅ phase3: 理解度確認・記録
- ✅ review_buffer
- ✅ portfolio_deliverables

**検出率 (Week 2-4平均)**:
- AI依頼プロンプト: 0% (意図的削除)
- 理解度チェックリスト: 100%
- 判定基準: 100%
- 3つ以上✅: 100%

**教育意図**: Week 2以降は、Week 1で学習したAI依頼品質向上手法を内在化し、明示的なプロンプトセクションなしで自律的にAI協働できるよう訓練。

---

### Template Type 3: 学習サマリー Format（学習要約形式）

**適用範囲**: D26, D30 (Week 5の一部)
**目的**: 復習・統合学習の簡素化

**構成**:
```markdown
#### Dxx: タイトル

**学習内容**:
- 統合学習項目1
- 統合学習項目2

**実装タスク**:
- タスク1
- タスク2

**ポートフォリオ成果物**:
- 成果物1
```

**特徴**:
- Phase 1/2/3構造なし
- 理解度チェックリストなし
- 直接的な実装タスクリスト

**検出率**:
- Phase 1/2/3: 0% (構造自体が存在しない)
- 理解度チェックリスト: 0%

**教育意図**: Week 1-4で習得した基礎技術の統合復習日。新規概念学習ではないため、詳細チェックリスト不要。

---

### Template Type 4: Challenge Format（自律チャレンジ形式）

**適用範囲**: Week 6 (D31-36) - 統合自律チャレンジ
**目的**: AI使用禁止による自律実装能力評価

**構成**:
```markdown
##### Dxx-xx: 統合自律チャレンジ

**総学習時間**: XXH (X日間)

**課題**: 新API統合実装
```python
要件:
1. Sync + Async両対応
2. エラー階層設計
3. pytest fixture作成
...

制約:
- AI使用完全禁止
- 公式ドキュメント閲覧OK
- 時間: XXH
```

**評価基準**:
```
完成度 (50%): ...
品質 (30%): ...
ドキュメント (10%): ...
自律度 (10%): ...

**合格: 75点以上**
```
```

**必須要素**: 課題、要件、制約、評価基準、合否判定

**検出率**:
- すべての標準テンプレート要素: 0% (意図的に異なる構造)

**教育意図**: Week 1-5の学習成果を、AI支援なしで実装できるか評価。標準3-Phaseテンプレートではなく、評価ルーブリック形式を採用。

---

### Template Type 5: Portfolio/YAML Format（ポートフォリオ・YAML形式）

**適用範囲**: Week 9-10 (D49-60) - ポートフォリオ最適化・応募準備
**目的**: ドキュメント品質、デモ環境、応募書類作成

**構成**:
```markdown
#### Dxx: タイトル

**総学習時間**: XH

**Phase 1: 設計・概念理解（XH）**:
```yaml
学習項目:
  - 項目1（XX Min）
    - 詳細1
  - 項目2（XX Min）
    - 詳細2
  - 理解度測定（15 Min）
  - バッファ（15 Min）
```

**Phase 2: 実装（XH）**:
```markdown
# 実装例
## セクション1
- 具体的実装コード例
```

**Phase 3: 理解度確認・記録（0.5H）**:
...

**復習バッファ（1H）**:
...
```

**特徴**:
- ✅ Phase 1/2/3構造あり
- ✅ YAML形式の学習項目（時間配分明記）
- ❌ 5項目チェックボックスなし
- ❌ 判定基準なし
- ✅ 具体的な実装コード例

**検出率 (Week 9-10平均)**:
- Phase 1/2/3: 100%
- 理解度チェックリスト: 0% (YAML学習項目に置き換え)
- 判定基準: 0%

**教育意図**: ポートフォリオ最適化フェーズでは、概念理解の自己評価より、具体的な実装成果物の品質向上を重視。YAML形式で時間配分を明示し、効率的な学習を促進。

## テンプレート使用ガイドライン

### 新規Day追加時の判断基準

| 学習内容タイプ | 推奨テンプレート | 理由 |
|---------------|----------------|------|
| 新規技術の基礎学習（Week 1） | Type 1: Full Standard | AI協働手法の教育必須 |
| 新規技術の基礎学習（Week 2以降） | Type 2: Standard (No AI) | 自律学習の促進 |
| 復習・統合学習 | Type 3: 学習サマリー | 詳細チェックリスト不要 |
| 自律実装評価 | Type 4: Challenge | 評価ルーブリック形式 |
| ポートフォリオ最適化 | Type 5: Portfolio/YAML | 実装例重視、時間配分明記 |
