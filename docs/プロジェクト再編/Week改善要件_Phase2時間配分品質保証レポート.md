### 3. カバレッジ計算ロジックへの影響評価

#### カバレッジ計算式（Week改善要件.md Line 172-197）

```python
# Week X全体での目標カバレッジ
weekX_target = [Week Xの終了時目標]  # 例: Week 1=39.5%
weekX_start = [Week Xの開始値]  # 例: Week 1=0%
weekX_total_impl_hours = [Week Xの実装時間]  # 例: Week 1=15h

# 時間あたりカバレッジ上昇率（実測値）
hourly_rate = (weekX_target - weekX_start) / weekX_total_impl_hours
# 例: Week 1では(39.5%-0%)/15h=2.63%/h

# Day別実装時間（Phase 2のみ）
day1_impl_hours = 2.5h  # Phase 2
```

#### Phase 2増加による影響

| 項目 | 改善前 | 改善後 | 差分 | 影響 |
|------|--------|--------|------|------|
| Phase 2時間/日 | 2.5h | 3.0h | +0.5h | 実装時間増加 |
| 週間実装時間 | 15h（2.5h×6日） | 18h（3.0h×6日） | +3h | 実装時間+20% |
| 時間あたり上昇率 | 2.63%/h | 2.19%/h | -0.44%/h | **効率-17%** |

**影響詳細**:
1. **Week 1カバレッジ目標への影響**
   - 目標: 39.5%（週終了時）
   - 改善前効率: 2.63%/h × 15h = 39.5%（達成可能）
   - 改善後効率: 2.19%/h × 18h = 39.4%（ほぼ達成、-0.1%誤差）
   - ✅ 結論: カバレッジ目標は維持可能（誤差範囲内）

2. **日次カバレッジ目標の再計算必要性**
   - 改善前: Day 1 → 6.6%（2.5h × 2.63%/h）
   - 改善後: Day 1 → 6.6%（3.0h × 2.19%/h）
   - ✅ 結論: 日次目標は維持可能（時間増加×効率低下が相殺）

3. **効率低下の原因分析**
   - 実装時間増加（+0.5h/日）により、単位時間あたりの成果が減少
   - 実測データ（2.63%/h）はPhase 2 (2.5h)時の効率
   - Phase 2 (3.0h)時の実測データなし → 効率仮定が必要

**修正提案**:
- カバレッジ計算ロジックの注記追加:
  ```yaml
  注記:
    - 時間あたり上昇率（2.63%/h）はPhase 2 (2.5h)時の実測値
    - Phase 2 (3.0h)に増加時は効率低下の可能性あり
    - 実測データ収集により再計算推奨
  ```

**結論**: ⚠️ **要検討**（カバレッジ目標は維持可能だが、効率仮定の妥当性要検証）
