#### 2.3.2 Day 44理解度確認問題（例）

**問題1: 概念説明（5点）**
```markdown
GitHub Actions cacheの`restore-keys`パラメータは何のために存在しますか？
`key`パラメータとの違いを含めて説明してください。

【採点基準】
5点: key完全一致時の優先順位 + restore-keys部分一致時のfallback仕組みを正確に説明
3点: keyとrestore-keysの違いを説明しているが、fallback仕組みが不明確
1点: keyとrestore-keysの存在を認識しているが、違いが説明できていない
0点: 誤った説明または無回答
```

**問題2: 設計判断（10点）**
```markdown
以下のシナリオで、Parallel Job実行とSequential Job実行のどちらを選択すべきか、
理由とともに説明してください。

シナリオ:
- Job A: pytest実行（5分）
- Job B: ruff/mypy実行（2分）
- Job C: Docker image build（10分）
- Job Cはtest/quality合格後のみ実行したい

【採点基準】
10点: Job A・B並列実行 + Job Cを`needs: [A, B]`で逐次実行の設計 + 理由説明
      （理由: test/quality独立、deploy依存関係、CI時間削減）
7点: 設計は正しいが、理由が部分的（例: CI時間削減のみ）
4点: 設計の方向性は正しいが、needs構文が不正確
0点: 誤った設計または無回答
```

**問題3: トラブルシューティング（10点）**
```markdown
GitHub Actions workflowがBuild time 5分超過でタイムアウトしました。

ログ:
```
Run uv sync
Installing dependencies...

Error: The operation was canceled.
```

【採点基準】
10点: 原因特定（cache未設定によるダウンロード時間超過） + 解決策3つ提案

7点: 原因特定は正しいが、解決策が2つのみ
4点: 原因特定が部分的（ダウンロード時間超過まで特定できているが、cache未設定に気づいていない）
0点: 誤った原因特定または無回答
```
