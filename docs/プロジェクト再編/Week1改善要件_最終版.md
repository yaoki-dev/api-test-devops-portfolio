### 1. Phase 1追加（3h/日）+ AI説明自動化

#### 問題
- **現状**: Phase 1が0.5h → 概念理解7%（原則2目標60% << 実績7%）
- **影響**: AI協働12原則 原則2「理解優先」違反
- **リスク**: 概念理解不足のまま実装 → 理解度30%未達

#### 解決策

**Phase 1構成**（3h/日）:
```yaml

  1. AI説明受領（10秒、自動化）:
     - Trigger 1検出: "学習開始"
     - AI説明自動生成:

        1. Why（何のための技術か）
        2. When（いつ使うのか）

  2. 自己学習（2.75h）:
     - 公式ドキュメント精読
     - ノート作成（Why/When/How整理）
     - サンプルコード試行

  3. 理解度測定（15分）:
     □ Why/When/How説明できる（各30秒）
     □ Pitfall 2つ挙げられる（1分）
     □ Best Practice適用できる（2分）
     □ コード例を自力修正できる（5分）

     判定:
       - 3つ以上✅ → Phase 2へ進む（目標30%達成）
       - 0-2つ✅ → AI再説明 → 自己学習継続
```

### 2. 時間配分修正（原則2準拠）

#### 問題
- **現状**: 実装7h vs 概念0.5h（概念比率7% << 原則2目標60%）
- **違反**: AI協働12原則 原則2「理解優先: 概念60% + AI協働25% + 判断10% + 実装5%」
- **影響**: 実装偏重 → 概念理解不足 → Layer 1（概念30%）未達成

#### 原則2「理解優先」の定義
```yaml
AI協働12原則 原則2:
  概念理解: 60%（Why/When/How/Pitfall/Best Practice）
  AI協働: 25%（AI支援下での実装・レビュー）
  判断: 10%（設計判断・トラブルシューティング）
  実装: 5%（純粋な実装作業）

目的:
  学習時間配分を実装中心から概念理解中心に転換し、
  真の理解を優先する時間配分を実現する。
```

#### 解決策

**時間配分変更**:
| フェーズ | 改善前 | 改善後 | 原則2目標 | 説明 |
|---------|--------|--------|----------|------|
| Phase 1 概念理解 | 0.5h (7%) | 3h (38%) | **60%** | **Why/When/How理解** |
| Phase 2 AI協働実装 | 5h (63%) | 3h (38%) | **25%** | **AI支援下の実装** |
| Phase 3 理解度確認 | 0.5h (7%) | 1h (13%) | **10%** | **自己評価・記録** |
| 復習バッファ | 0h | 1h (13%) | - | **品質ゲート対応** |
| **合計** | **8h** | **8h** | **100%** | - |

**注記**:
- 改善後も原則2目標（60%）に未達（38%）だが、8h/日制約内での最適化
- Week 2以降で段階的に60%に近づける（実装効率化により概念時間増加）

### 3. AI協働率80%再定義

#### 問題
- **誤解**: AI協働率80% = "AIが80%代理実装" → 原則1「主体的実践」違反
- **正解**: AI協働率80% = "学習時間の80%でAI支援活用"（Phase 1 AI説明 + Phase 2 AI協働実装）
- **現状**: AI協働率測定方法が未定義 → Trigger 3で手動入力必要

#### 解決策

**AI協働率の再定義**:
```yaml
AI協働率の正しい理解:
  定義: 学習時間のうちAI支援を活用する時間の割合

  Week 1-2例（AI協働率80%）:
    Phase 1（3h）: AI説明受領 + 自己学習 = AI支援あり
    Phase 2（3h）: AI協働実装 = AI支援あり
    Phase 3（1h）: 理解度確認・記録 = AI支援なし
    復習バッファ（1h）: 品質ゲート修正 = AI支援あり

    AI支援時間: 3h + 3h + 1h = 7h
    総学習時間: 8h
    AI協働率: 7h / 8h = 87.5% ≈ 80%目標達成

  原則1「主体的実践」遵守:
    - ユーザー実装 → AI支援（AIが代理実装ではない）
    - AI説明 → ユーザー理解確認 → ユーザー実装
```

**Week別AI協働率目標**（Week固定値、手動測定廃止）:
```python
def get_ai_collab_rate_target(current_week: int) -> int:
    """
    Week番号からAI協働率目標を取得

    Args:
        current_week: 現在のWeek番号（1-10）

    Returns:
        AI協働率目標（%）
    """
    targets = {
        1: 80, 2: 80,  # Week 1-2: 80%（AI説明中心）
        3: 75, 4: 75,  # Week 3-4: 75%
        5: 70, 6: 70,  # Week 5-6: 70%
        7: 65, 8: 65,  # Week 7-8: 65%
        9: 60, 10: 60  # Week 9-10: 60%（自律実装中心）
    }
    return targets.get(current_week, 70)
```

**Trigger 3改善**: 手動入力削除（既存仕様v2.1を維持）

### 6. 復習バッファ追加

#### 問題
- **品質ゲート失敗時の復旧時間なし**: pytest/ruff/mypy失敗時の修正時間が未計上
- **実測データ**: Week 5-6で品質ゲート失敗時の修正に平均6-7h/週必要
- **スケジュールリスク**: 復旧時間不足 → 翌日へ遅延 → 連鎖遅延

#### 解決策

**復習バッファ設定**:
```yaml
毎日の復習バッファ:
  Day 1-5: 1h/日（合計5h）
  Day 6: 2h（週次調整バッファ）
  合計: 7h/週

用途:
  1. 品質ゲート失敗時の即時修正（最優先）
```
