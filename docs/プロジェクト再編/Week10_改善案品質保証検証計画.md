#### タイムバッファ分析

| 日次 | 計画時間 | 上限時間 | バッファ | リスク評価 |
|------|---------|---------|---------|-----------|
| Day 55-56 | 7H/日 | 7H/日 | 0H | 🔴 Critical: バッファゼロ |
| Day 57-58 | 7H/日 | 7H/日 | 0H | 🔴 Critical: バッファゼロ |
| Day 59 | 7H | 7H | 0H | 🔴 Critical: バッファゼロ |
| Day 60 | 7H | 7H | 0H | 🔴 Critical: バッファゼロ |

**発見事項（🔴 Critical）**:
- 全日でタイムバッファゼロ → 遅延時の復旧不可能
- 週全体でもバッファゼロ → 42H制約厳守必須

**リスク緩和策**:
1. **Day 60にバッファ機能統合**:
   - 応募実行 + 振り返り（7H）→ 応募実行5H + 振り返り1H + バッファ1H

2. **タイムトラッキング厳格化**:
   - 各Phase開始・終了時刻を記録
   - Phase超過時は即座に次Phaseを縮小調整

3. **オプショナルタスクの明示**:
   - 必須タスク vs オプショナルタスクを明確化
   - 時間不足時はオプショナルをスキップ

### 2.2 応募品質と応募数の両立可能性

#### トレードオフ分析

| 応募数 | 応募品質 | 所要時間 | 獲得確率（推定） | リスク評価 |
|--------|---------|---------|-----------------|-----------|
| 10件/週 | 90%品質 | 21H/週（2.1H/件） | 85%+ | ✅ 理想的（時間制約厳しい） |
| 20件/週 | 80%品質 | 28H/週（1.4H/件） | 75-80% | ✅ バランス良好 |
| 30件/週 | 70%品質 | 35H/週（1.17H/件） | 65-70% | ⚠️ 品質低下リスク |
| 40件/週 | 60%品質 | 42H/週（1.05H/件） | 50-60% | 🔴 品質不足 |

**分析結果**:
- 応募数20件/週が最適バランス点
- 応募品質80%、獲得確率75-80%を両立可能
- 所要時間28H/週 → Week 10の42H制約内で実現可能

**推奨戦略**:
```yaml
Week 10 (Day 55-60): 応募準備 + 初回応募10件（21H）
Week 10-B (Day 61-67): 追加応募20件（28H）

累計応募: 30件（Week 10 + 10-B）
累計時間: 49H（Week 10: 42H + Week 10-B: 7H）
応募品質: 80%維持
獲得確率: 75-80%（推定）
```

**50件達成への調整案**:
- Week 10-B（Day 61-67）の応募数を20件 → 30件に増加
  - 所要時間: 28H → 35H（5H/日 × 7日）
  - 応募品質: 80% → 75%（-5pt低下）
  - 獲得確率: 75-80% → 70-75%（-5pt低下）

**リスク評価**:
- 30件/週は応募品質70%まで低下 → 獲得確率65-70%に低下リスク
- 50件達成には品質トレードオフが必要

```python
# 獲得確率80%の統計的検定
from scipy.stats import binom

def validate_80_percent_claim(
    applications: int,  # 応募数
    acquisitions: int,  # 獲得数
    target_prob: float = 0.80,  # 目標獲得確率
    alpha: float = 0.05,  # 有意水準
) -> dict:
    """
    獲得確率80%主張の統計的検定

    帰無仮説: 獲得確率 = 80%
    対立仮説: 獲得確率 ≠ 80%

    Args:
        applications: 応募数
        acquisitions: 獲得数
        target_prob: 目標獲得確率
        alpha: 有意水準

    Returns:
        検定結果（p値、判定、信頼区間）
    """
    # 二項検定
    p_value = binom.cdf(acquisitions, applications, target_prob)

    # 95%信頼区間
    from statsmodels.stats.proportion import proportion_confint
    ci_low, ci_high = proportion_confint(
        acquisitions, applications, alpha=alpha, method='wilson'
    )

    # 判定
    if p_value < alpha:
        decision = "棄却（獲得確率80%ではない）"
    else:
        decision = "棄却できず（獲得確率80%の可能性あり）"

    return {
        "応募数": applications,
        "獲得数": acquisitions,
        "実測獲得率": acquisitions / applications,
        "p値": p_value,
        "判定": decision,
        "95%信頼区間": (ci_low, ci_high),
    }


# シミュレーション例
# ケース1: 30件応募、24件獲得（獲得率80%）
case1 = validate_80_percent_claim(applications=30, acquisitions=24)
print(f"ケース1: {case1}")

# ケース2: 30件応募、20件獲得（獲得率67%）

print(f"ケース2: {case2}")

# ケース3: 50件応募、40件獲得（獲得率80%）

print(f"ケース3: {case3}")
```

- 応募数30件以上で95%信頼区間が±10%以内に収束
