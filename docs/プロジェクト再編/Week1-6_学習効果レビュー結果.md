#### 習熟度計算ロジックの妥当性（中）

**問題**: `mastery_level = (actual_hours / estimated_hours) * 85` が単純すぎる

**現状の問題点**:
1. **時間のみで評価**: 学習品質・理解度が反映されない
2. **85%固定**: 個別スキルの難易度差が考慮されない
3. **線形計算**: 習熟度は非線形（初期は早く、後期は遅い）

**推奨改善策**:
```yaml
# 修正前
mastery_level = (actual_hours / estimated_hours) * 85

# 修正後（多次元評価）
mastery_level = weighted_average([
  time_factor * 0.3,      # 学習時間達成率（30%）
  test_score * 0.5,       # 理解度テストスコア（50%）
  implementation_quality * 0.2  # 実装品質（20%）
])

# 計算例（Day 1終了時）
time_factor = (5h / 7h) * 100 = 71%
test_score = 60/75 * 100 = 80%
implementation_quality = (pytest合格: 100%, coverage目標達成: 90%, ruff/mypy合格: 100%) / 3 = 97%

mastery_level = 0.3*71 + 0.5*80 + 0.2*97
              = 21.3 + 40 + 19.4
              = 80.7%（要復習 → 85%未満）
```
