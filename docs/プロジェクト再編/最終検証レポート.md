## テンプレート形式分類

学習計画は、教育目的に応じて5種類の異なるテンプレート形式を使用します。

### Type 1: Full Standard Template（完全標準テンプレート）

**適用範囲**: Week 1 (D1-6)
**目的**: AI協働学習の基礎教育、AI依頼品質向上

**必須要素（12個）**:
- ✅ day_header
- ✅ total_learning_time
- ✅ coverage_goal
- ✅ phase1: AI説明・概念理解
- ✅ understanding_checklist (5項目チェックボックス)
- ✅ judgment_criteria (判定基準)
- ✅ criteria_3plus (3つ以上✅基準)
- ✅ phase2: AI協働実装
- ✅ ai_request_prompt (AI依頼プロンプトセクション)
- ✅ phase3: 理解度確認・記録
- ✅ review_buffer
- ✅ portfolio_deliverables

**教育意図**: Week 1では、AI協働の基礎（プロンプト品質、理解度自己評価）を徹底教育するため、全要素を含む最も詳細なテンプレートを使用。

---

### Type 2: Standard Template (No AI Prompt)（標準テンプレート・AI依頼セクションなし）

**適用範囲**: Week 2-4 (D7-24), Week 8 (D43-48の一部)
**目的**: 自律的学習への移行、AI依存度低減

**必須要素（11個）**:
- ✅ day_header
- ✅ total_learning_time
- ✅ coverage_goal
- ✅ phase1: AI説明・概念理解
- ✅ understanding_checklist (5項目チェックボックス)
- ✅ judgment_criteria (判定基準)
- ✅ criteria_3plus (3つ以上✅基準)
- ✅ phase2: AI協働実装
- ❌ ai_request_prompt (意図的に削除)
- ✅ phase3: 理解度確認・記録
- ✅ review_buffer
- ✅ portfolio_deliverables

**教育意図**: Week 1で学習したAI依頼品質向上手法を内在化し、明示的なプロンプトセクションなしで自律的にAI協働できるよう訓練。

---

### Type 3: 学習サマリー Format（学習要約形式）

**適用範囲**: D26, D30 (Week 5), D37, D38, D42 (Week 7)
**目的**: 復習・統合学習の簡素化

**構成**:
```markdown
#### Dxx: タイトル

**学習内容**:
- 統合学習項目1
- 統合学習項目2

**実装タスク**:
- タスク1
- タスク2

**ポートフォリオ成果物**:
- 成果物1
```

**特徴**:
- Phase 1/2/3構造なし
- 理解度チェックリストなし
- 直接的な実装タスクリスト

**教育意図**: Week 1-4で習得した基礎技術の統合復習日。新規概念学習ではないため、詳細チェックリスト不要。

---

### Type 4: Challenge Format（自律チャレンジ形式）

**適用範囲**: Week 6 (D31-36) - 統合自律チャレンジ
**目的**: AI使用禁止による自律実装能力評価

**構成**:
```markdown
##### Dxx-xx: 統合自律チャレンジ

**総学習時間**: XXH (X日間)

**課題**: 新API統合実装
```python
要件:
1. Sync + Async両対応
2. エラー階層設計
3. pytest fixture作成
...

制約:
- AI使用完全禁止
- 公式ドキュメント閲覧OK
- 時間: XXH
```

**評価基準**:
```
完成度 (50%): ...
品質 (30%): ...
ドキュメント (10%): ...
自律度 (10%): ...

**合格: 75点以上**
```
```

**教育意図**: Week 1-5の学習成果を、AI支援なしで実装できるか評価。標準3-Phaseテンプレートではなく、評価ルーブリック形式を採用。

---

### Type 5: Portfolio/YAML Format（ポートフォリオ・YAML形式）

**適用範囲**: Week 9-10 (D49-60) - ポートフォリオ最適化・応募準備
**目的**: ドキュメント品質、デモ環境、応募書類作成

**構成**:
```markdown
#### Dxx: タイトル

**総学習時間**: XH

**Phase 1: 設計・概念理解（XH）**:
```yaml
学習項目:
  - 項目1（XX Min）
    - 詳細1
  - 項目2（XX Min）
    - 詳細2
  - 理解度測定（15 Min）
  - バッファ（15 Min）
```

**Phase 2: 実装（XH）**:
```markdown
# 実装例
## セクション1
- 具体的実装コード例
```

**Phase 3: 理解度確認・記録（0.5H）**:
...

**復習バッファ（1H）**:
...
```

**特徴**:
- ✅ Phase 1/2/3構造あり
- ✅ YAML形式の学習項目（時間配分明記）
- ❌ 5項目チェックボックスなし
- ❌ 判定基準なし
- ✅ 具体的な実装コード例

**教育意図**: ポートフォリオ最適化フェーズでは、概念理解の自己評価より、具体的な実装成果物の品質向上を重視。YAML形式で時間配分を明示し、効率的な学習を促進。

## 検証スクリプトの限界

### 検出精度の課題

検証スクリプト（`verify_learning_plan_structure.py`）は、標準的なType 1/2形式には高精度で動作しますが、以下の場合に誤検出が発生します：

#### 1. Type 3 学習サマリー形式の誤検出

**問題**: Type 3形式は意図的に以下の要素を省略しているが、スクリプトは「欠落」として検出
- 理解度チェックリスト（直接タスクリストで代替）
- 判定基準（復習日のため評価不要）
- Phase 1/2/3構造（または簡素化版）

**影響**: D26, D30, D37, D38, D42が「要確認」としてフラグされる

**対策**: テンプレート仕様書にType 3形式の検出率低下が正常である旨を明記

---

#### 2. Type 4 Challenge形式の誤検出

**問題**: Type 4形式は評価ルーブリック形式を使用し、標準Phase構造を使用しない
- Phase 1/2/3構造なし（課題・要件・評価基準に置き換え）
- 理解度チェックリストなし（評価ルーブリックで代替）

**影響**: D34-36が「要確認」としてフラグされる（検出率0%）

**対策**: Week 6は意図的な構造変更として受け入れ

---

#### 3. Type 5 Portfolio/YAML形式の誤検出

**問題**: Type 5形式はYAML学習項目を使用し、チェックリスト形式を使用しない
- 5項目チェックボックスなし（YAML学習項目で代替）
- 判定基準なし（時間配分管理重視）

**影響**: D49-60が「要確認」としてフラグされる（検出率低下）

**対策**: Week 9-10はポートフォリオ最適化期間として構造変更を受け入れ

---

#### 4. Phase 3セクションの誤検出

**問題**: 一部のDayでPhase 3セクションが存在するが、正規表現が検出できない
- 実際には存在: `**Phase 3: 理解度確認・記録**`
- 検出失敗: 正規表現の厳格性による（例: 空白文字の違い、改行の有無等）

**影響**: D43-47の一部でPhase 3が「欠落」として検出

**対策**: 正規表現の柔軟性向上（オプション）、または検出率低下を許容
