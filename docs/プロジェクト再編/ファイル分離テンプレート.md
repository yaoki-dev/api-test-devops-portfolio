# Phase 1・Phase 2 コンテンツ分離ガイドライン

*最終更新: 2025年11月02日*

## 📋 目的

**学習プランファイル**（`10週ハイブリッドプラン_日次詳細学習スケジュール.md`）と**ポートフォリオ戦略ファイル**（`ポートフォリオ戦略.md`）の役割を明確化し、コンテンツの重複を排除する。

### なぜ分離が必要か

**現状の問題**:
- 学習プランに実装コード例・AI プロンプトが含まれている
- ポートフォリオ戦略に学習時間配分・理解度チェックリストが含まれている
- 両ファイルの責任範囲が曖昧で、情報が散在している

**分離後のメリット**:
- ✅ 学習プラン: **「何を」「いつ」「どう学ぶか」**に専念
- ✅ ポートフォリオ戦略: **「何を」「どう実装するか」**に専念
- ✅ ファイルサイズ削減（40k+ chars → 20k chars 程度）
- ✅ メンテナンス性向上（更新箇所の明確化）

"""
AsyncAPIClient を以下の要件で実装してください:

   - async Context Manager 対応
   - BaseAPIClient との設計一貫性

3. セキュリティ:
   - タイムアウト設定必須
   - リトライロジック実装

4. パフォーマンス:
   - コネクションプール活用

**実装例**:
```python
from typing import Optional, Any
import httpx

class AsyncAPIClient:
    """非同期APIクライアント基底クラス

    httpx.AsyncClientをラップし、async Context Manager対応で
    非同期HTTP通信を実装する。

    Attributes:
        base_url: APIのベースURL
        timeout: タイムアウト秒数（デフォルト30秒）
        _client: httpx.AsyncClientインスタンス
    """

    def __init__(self, base_url: str, timeout: float = 30.0):
        self.base_url = base_url
        self.timeout = timeout
        self._client: Optional[httpx.AsyncClient] = None

    async def __aenter__(self):
        """async Context Manager - 入口

        httpx.AsyncClientインスタンスを生成し、
        非同期HTTP接続を確立する。
        """
        self._client = httpx.AsyncClient(
            base_url=self.base_url,
            timeout=self.timeout
        )
        return self

    async def __aexit__(self, exc_type, exc_val, exc_tb):
        """async Context Manager - 出口

        httpx.AsyncClientインスタンスをクローズし、
        リソースを解放する。
        """
        if self._client:
            await self._client.aclose()
```

## 🔍 エッジケース対応

### Q1: Phase 2の時間配分詳細はどこに記載？

**回答**: 学習プランに記載（学習スケジュール管理のため）

**例**:
```markdown
**Phase 2: AI協働実装 (3H)** → 理解度60%目標
  - 実装詳細は `@ポートフォリオ戦略.md#W2-D7` を参照
  - AsyncAPIClient実装 [70 Min]
  - async Context Manager実装 [40 Min]
  - pytest-asyncio導入 [30 Min]
  - 理解度確認テスト作成 [40 Min]
```

### Q2: カバレッジ目標はどこに記載？

**回答**: ポートフォリオ戦略のみに記載

- **ポートフォリオ戦略**: 各Task単位で記載（「Task 2.1: カバレッジ目標 44.58%」）
- **学習プランから参照**: 実装詳細を参照する際に確認

**理由**: カバレッジ目標は実装成果メトリクスであり、学習目標ではない。学習プランではポートフォリオ戦略への参照リンクにより、必要時に確認可能。

### Q3: 週次サマリーはどこに記載？

**回答**: 両ファイルに記載（異なる視点）

- **学習プラン**: 学習効果サマリー（「習得スキル」「理解度達成率」）
- **ポートフォリオ戦略**: 実装成果サマリー（「完成タスク数」「カバレッジ推移」「テスト数推移」）
