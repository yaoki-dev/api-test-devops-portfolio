#### ステップ2: 優先度付け (0.2h)

**優先度スコアリング**:

スコア = (重要度 × 3) + (理解不足度 × 2)

**重要度** (1-5点):
- 5点: Week核心概念（例: Async/Await基礎）
- 4点: 頻出パターン（例: pytest-asyncio）
- 3点: 応用技術（例: Exponential backoff）
- 2点: 補助的概念（例: ログ設定）
- 1点: Nice-to-have（例: docstring装飾）

**理解不足度** (1-5点):
- 5点: 全く理解できていない（説明できない）
- 4点: 部分的理解（コード見ないと実装できない）
- 3点: 基本理解（コード見れば実装できる）
- 2点: 概ね理解（自力実装70%可能）
- 1点: 十分理解（自力実装90%可能）

**例**:
- Async/Await基礎: (5 × 3) + (4 × 2) = 23点 → **最優先**
- pytest-asyncio: (4 × 3) + (3 × 2) = 18点 → 優先
- docstring装飾: (1 × 3) + (2 × 2) = 7点 → 後回し

**成果物**: スコア順つまずき箇所リスト（上位3項目に絞る）

**例（Async/Await基礎復習）**:
```python
# 最小実装例（5分で作成）
import asyncio

async def fetch_user(user_id: int):
    await asyncio.sleep(0.1)  # 非同期処理シミュレーション
    return {"id": user_id, "name": f"User {user_id}"}

async def main():
    # 並行処理テスト
    users = await asyncio.gather(
        fetch_user(1),
        fetch_user(2),
        fetch_user(3)
    )
    print(users)  # 3ユーザー同時取得確認

# 実行
asyncio.run(main())
```
