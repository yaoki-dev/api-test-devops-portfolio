  2. 根本原因分析記載:
  問題: 内部try-exceptで raise された APIHTTPError が
       外部catch-allハンドラーに捕捉され、リトライが継続される

  影響: 4xxクライアントエラーがリトライされる
       （コメントでは「リトライしない」と記載）

  修正案:
  - catch-allハンドラーを内部tryに移動
  - または特定の例外型のみキャッチ
  3. テスト参照:
    - tests/unit/test_async_client_error_handling.py
