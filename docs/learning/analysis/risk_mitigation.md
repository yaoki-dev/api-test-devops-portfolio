### 期待値調整テクニック

#### プロジェクト開始前
1. **技術査定**: 要件の複雑度評価、実現可能性判定
2. **段階的提案**: Phase別実装、リスク分散
3. **品質基準設定**: 測定可能な成果指標合意

#### プロジェクト進行中
1. **定期報告**: 週次進捗、課題早期発見
2. **透明性確保**: 困難な点の率直な報告
3. **解決策提示**: 問題発見時の具体的対応案

## 🔒 品質保証とリスク管理体制

### 3層品質保証システム

#### Layer 1: 自動化品質ゲート
```bash
# 必須品質チェック（自動実行）
make quality  # ruff + mypy + bandit
make coverage # 85%カバレッジ確認
make security # OWASP準拠チェック
```

#### Layer 2: AI協働レビュー
```python
# Claude Code review process
REVIEW_PROCESS = {
    "コード品質": "自動レビュー + 改善提案",
    "アーキテクチャ": "設計パターン検証",
    "セキュリティ": "脆弱性スキャン + 対策",
    "パフォーマンス": "ボトルネック特定 + 最適化"
}
```

#### Layer 3: 外部専門家レビュー
- **発動条件**: 複雑度スコア8以上、未経験技術領域
- **レビュー範囲**: アーキテクチャ、セキュリティ、パフォーマンス
- **コスト**: プロジェクト予算の5-10%

### リスク管理プロトコル

#### リスク分類と対応策
| リスク Level | 対応策 | エスカレーション |
|-------------|--------|-----------------|
| **Low (1-3)** | 自己解決 | 不要 |
| **Medium (4-6)** | AI協働 + 調査 | 日次報告 |
| **High (7-8)** | 外部相談 | 即座報告 |
| **Critical (9-10)** | 専門家投入 | 緊急対応 |
